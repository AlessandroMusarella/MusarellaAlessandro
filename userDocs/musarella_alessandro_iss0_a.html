Introduction

Our motto:
there is no code without a project, no project without problem analysis and no problem without requirements.

Requirements

Design and build a software system that makes a robot is able to walk along the boundary of a rectangular, empty room.

Requirement analysis

From the requirements, it follows that:
  1. room: a defined space,rectangular in shape, with no impediment for the mobility
  2. boundary: the perimeter of the room
  3. robot: an entity capable of moving
  4. walk: the act of moving across a space

User story

The user places the robot in the HOME position (facing south) and then activate a sistem which sends movement commands to the robot (via wifi network).
The user cannot interrupt the sistem execution: it must terminate autonomously at the end of the task.

At the end of the execution, the robot is expected to have completed the path showed in the image (only once).

Test plans

It must be verified that the robot follows the expected path.
The correctness of the path must be verified via software, without the human intervention.

Problem analysis

Relevant aspects

  1. We must build a distributed system made by two macro-components:
    • the robot provided by the client
    • our application (boundaryWalk) that sends commands to the robot in order to meet the requirements
  2. The robot can be controlled via network by two different means, as described in virtualRobot2021:
    • sending messages to the 8090 port with the HTTP POST protocol
    • sending messages to the 8091 port with a websocket
  3. As there are multiple libraries for various programming language to support these two operations, we do not detect a significant abstraction gap
  4. We estimate that a first functional prototype can be realized at most in a work day

Main problems

Interaction abstraction

The software system should be indipendent from the communication protocol used for the interaction with te WEnv.
The designer could use some design pattern to make the application level as indipendent as possible from the communication procolol, which can be selected as a configuration parameter.

Functional testplans

In order to control that the robot completed the walk around the boundaries of the room, it is necessary to check if the robot moves forward and makes a turn, always in the same direction, for 4 consecutive times.
